## 1. Структура проекта (Архитектура)
При создании новых модулей или страниц строго следуй структуре разделения ответственности (Separation of Concerns):
* **Разметка (`.html`):** Только структура и семантика. Никаких стилей в тегах и инлайновых обработчиков (`onclick=""`).
* **Стили (`.css`):** Только внешний вид и анимации. Вся динамика — через переключение CSS-классов.
* **Логика (`.js`):** Только поведение. Добавление событий через `addEventListener`.

Пример файловой структуры модуля:
```text
📁 components/widget/
├── 📄 widget.html     # Чистая разметка компонента
├── 📄 widget.css      # Изолированные стили (БЭМ)
└── 📄 widget.js       # Логика и инициализация
```

## 2. Стандарты HTML (Семантика и Доступность)
* **Семантика:** Используй правильные теги (`<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`, `<button>`). Не заменяй всё на `<div>`.
* **Формы:** Всегда связывай `<label>` и `<input>` через атрибут `id` и `for`. Кнопка отправки формы должна иметь `type="submit"`.
* **Доступность (a11y):** Обязательно указывай `alt` для картинок. Используй `aria-*` атрибуты для динамических элементов (например, `aria-expanded="false"` для дропдауна).

## 3. Стандарты CSS (Архитектура и Стилизация)
* **Методология:** Используй упрощенный БЭМ (Блок-Элемент-Модификатор) для именования классов (например, `.card`, `.card__title`, `.card--featured`).
* **Верстка:** Для раскладки используй **Flexbox** и **CSS Grid**. Никаких флоатов (`float`) или табличной верстки для макетов.
* **Переменные:** Выноси цвета, шрифты и отступы в CSS-переменные внутри `:root`.
* **Адаптивность:** Пиши адаптивные стили через `@media` запросы (рекомендуется подход Mobile First).
* **Сброс стилей:** Убедись, что используется базовый `box-sizing: border-box`.

## 4. Стандарты JavaScript (Modern ES6+)
* **Переменные:** Запрещено использовать `var`. Используй `const` по умолчанию, и `let`, только если значение меняется.
* **Выбор элементов:** Используй `document.querySelector()` и `document.querySelectorAll()` вместо устаревших `getElementById`.
* **Манипуляции с DOM:** 
  * Для изменения текста используй `textContent`, а не `innerHTML` (защита от XSS).
  * Для управления стилями добавляй/удаляй классы: `element.classList.add('is-active')`. Не меняй `element.style` напрямую, кроме динамических координат.
* **Асинхронность:** Используй синтаксис `async/await` и `fetch()` для работы с API. Избегай старых `XMLHttpRequest` и цепочек `.then()`, если они ухудшают читаемость.
* **Изоляция кода:** Оборачивай код в модули (`import/export`) или используй IIFE / событие `DOMContentLoaded`, чтобы не засорять глобальную область видимости (`window`).

## 5. Чек-лист проверки (Definition of Done)
Перед завершением задачи убедись, что:
* [ ] В консоли браузера нет ошибок JS.
* [ ] HTML валиден и не содержит незакрытых тегов.
* [ ] Страница корректно отображается на мобильных экранах (адаптивность).
* [ ] Код работает без подключения внешних библиотек (jQuery, Bootstrap и т.д.).
